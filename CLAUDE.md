# Talk-To-My-Lawyer (TTML) Project Guidelines for Claude Code

This `CLAUDE.md` file provides essential context and architectural guidelines for Claude Code when working on the TTML application. It ensures that all AI-assisted development adheres to our core principles of security, consistency, and maintainability.

## Core Architectural Principles

All modifications and code generations must respect the following fundamental patterns:

1.  **Mandatory Attorney Review**: Every legal letter generated by the AI pipeline MUST be reviewed and approved by an attorney. The `ai_draft` version is immutable and serves as an audit trail.
2.  **Strict Status Machine**: Letter requests follow a strict 15-status sequential path. All status transitions must be validated against `ALLOWED_TRANSITIONS` in `shared/types.ts`.
3.  **Role-Based Access Control (RBAC)**: Access is strictly gated by tRPC middleware. Always verify `userRole` before allowing actions.
4.  **Super Admin Whitelist**: Super admin access is hard-coded in `server/supabaseAuth.ts` and cannot be granted via UI or API. Non-whitelisted users are silently demoted.
5.  **Attorney Promotion Flow**: Attorneys are promoted only by super admins, and active subscribers cannot be promoted.
6.  **Session Refresh**: Role changes take effect immediately due to `invalidateUserCache()` and frontend `refetchOnWindowFocus`.
7.  **Payment Gate**: Full letter content is locked (`generated_locked` status) until payment is confirmed, with server-side truncation and frontend blurring.

## Systematic Analysis Workflow

Before making any changes, consult the following:

*   **Schema**: `drizzle/schema.ts`
*   **State Transitions**: `shared/types.ts` (`ALLOWED_TRANSITIONS`)
*   **Role Verification**: `server/routers/` (tRPC procedure guards)
*   **AI Pipeline**: `server/pipeline/orchestrator.ts`
*   **Auditability**: `logReviewAction` for `review_actions` table
*   **Side Effects**: Emails, payments, RAG, training data
*   **Super Admin Whitelist**: `server/supabaseAuth.ts`
*   **Entitlements**: Atomic usage claim for letter creation
